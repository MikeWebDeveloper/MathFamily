# ParkMath SEO Fixes — Batch 1 (Quick Wins + P0) — Design Spec

*Ship the highest impact-to-effort SEO/code fixes from the 2026-06-12 audit
(`docs/seo-audit/2026-06-12-seo-code-audit.md`): canonical/OpenGraph metadata, security headers,
Organization + NewsArticle schema, an LCP font-preload fix, answer-first index summaries, self-citing

ParkMath only; RoamMath inherits the shared-package fixes (geo/ui) but its app isn't otherwise touched.*

## Intent

The audit graded the site well-built; the gaps cluster into canonical/OG, entity/price schema, and
security-headers. This batch closes the four P0s plus the cheapest high-value P1s, **without
regressing the many things already done right** (see the audit's "Already excellent — do NOT touch").

## Guardrails (do NOT change — verified correct)

Apex→www/trailing-slash 308s; real-404 + `dynamicParams=false`; **no `hreflang`** (single-locale —
adding it is wrong); the sitemap/robots/`llms.txt`; existing valid schema; **no fabricated reviews**;
**no raster images exist** (only the `aria-hidden` `UkMap` SVG) — do not add alt/images.

## Units

### A. Canonical + OpenGraph metadata *(P0-1, quick wins 1, 2, 10)*

- **Canonical:** `metadataBase` is already set. Add `alternates.canonical` everywhere:
  - Root `apps/parkmath/app/layout.tsx` metadata: `alternates: { canonical: "/", types: { "application/rss+xml": "/news/feed.xml" } }` (keep the RSS type).
  - Every dynamic/static route's `generateMetadata`/`metadata` sets `alternates.canonical` to its **exact path** (e.g. `/drop-off-charges/${slug}`, `/airport-parking/${slug}`, `/airport-parking/${slug}/${duration}`, `/airport-lounges/${slug}`, `/news`, `/news/${id}`, and the index pages `/drop-off-charges`, `/airport-parking`, `/airport-lounges`). Next resolves these against `metadataBase` and **auto-emits `og:url`** from the canonical.
- **OpenGraph:** add an `openGraph` block — root: `{ type: "website", siteName: "ParkMath", locale: "en_GB" }`; per `news/[id]`: `{ type: "article", publishedTime: publishedAt, modifiedTime: verifiedAt }`. (Twitter card is auto-generated by Next from OG — do **not** add it manually.)
- **Titles:** shorten the overflowing drop-off template `apps/parkmath/app/drop-off-charges/[airport]/page.tsx` to `${name} drop-off charge 2026 — fee, limit & free alternative` (~60 chars); trim the lounge-index / lounge / parking titles that exceed ~60.
- **Verification (live/preview, not unit):** fetched HTML contains exactly one `<link rel="canonical" href="https://www.parkmath.co.uk/<path>">` per page and a matching `og:url`; titles ≤ ~60 chars.

### B. Security headers *(P0-2, quick win 3)*

Add `async headers()` to `apps/parkmath/next.config.ts` applying to all routes:
- `X-Content-Type-Options: nosniff`
- `Referrer-Policy: strict-origin-when-cross-origin`
- `X-Frame-Options: DENY`
- `Permissions-Policy: camera=(), microphone=(), geolocation=(), interest-cohort=()`
- `Strict-Transport-Security: max-age=63072000; includeSubDomains; preload`
- **CSP** (pragmatic for a fully-static site with inline JSON-LD + count-up islands; nonces would force
  dynamic rendering, so `'unsafe-inline'` for script/style is the accepted trade-off — frame-ancestors
  and the origin allowlist still harden it):
  ```
  default-src 'self';
  script-src 'self' 'unsafe-inline' https://static.cloudflareinsights.com https://www.dwin1.com;
  style-src 'self' 'unsafe-inline' https://fonts.googleapis.com;
  font-src 'self' https://fonts.gstatic.com;
  img-src 'self' data: https:;
  connect-src 'self' https://cloudflareinsights.com https://static.cloudflareinsights.com;
  frame-ancestors 'none'; base-uri 'self'; object-src 'none'; upgrade-insecure-requests
  ```
  (No `form-action` directive in v1 — the MailerLite email-capture posts cross-origin and its exact
  domain isn't pinned here; tightening `form-action` is a P2 follow-up once that origin is confirmed.)
- **MANDATORY verification before merge:** deploy the branch to a Vercel **preview**, open it, and
  confirm the browser console shows **zero CSP violations**, and that the Cloudflare beacon, the (future)
  AWIN tag, Google Fonts, the JSON-LD blocks, and the calculators all still work. Do not merge a CSP
  that throws violations.

### C. Organization + NewsArticle schema *(P0-3, P0-4, quick wins 5, 6)*

- **`packages/geo/src/builders.ts`:** add `organizationLd({ siteUrl, name, logoUrl, sameAs })` →
  `{ "@context":"https://schema.org", "@type":"Organization", "@id": \`${siteUrl}/#organization\`, name, url: siteUrl, logo: { "@type":"ImageObject", url: logoUrl }, sameAs }`.
  - `logoUrl` = the existing root OG image route `${siteUrl}/opengraph-image` (a real 1200×630 PNG — valid for `logo`/`image`). `sameAs` = `[]` for now (no social profiles yet) — omit the key if empty.
- **Render Organization once** in `apps/parkmath/app/layout.tsx` via `<JsonLd data={organizationLd(...)} />`.
- **Fix `newsArticleLd()`** — add `image: [\`${siteUrl}/opengraph-image\`]`, `author: { "@type":"Organization", "@id": \`${siteUrl}/#organization\` }`, and make `publisher` reference `{ "@type":"Organization", "@id": \`${siteUrl}/#organization\` }` (so the publisher carries a `logo` via the Organization node). Keep existing fields.
- **`@id`-reference** the Organization from `datasetLd`'s `creator` too (if it currently inlines a bare publisher).
- **Testing (unit, `packages/geo`):** `organizationLd` returns the right `@type`/`@id`/`logo` shape; `newsArticleLd` now includes `image`, `author`, and a publisher `@id`. Follows the existing geo builder-test pattern.

### D. Answer-first index summaries + self-citing FAQ *(P1-3, P1-4, quick wins 7, 8)*

- **Index summaries:** on `apps/parkmath/app/drop-off-charges/page.tsx` (and the parking + lounge
  index pages), add a server-rendered answer sentence + an `<h2>` intro **above** the table, derived
  from the already-loaded `records`:
  - drop-off: free-count, cheapest paid (airport + £), dearest (airport + £).
  - parking/lounge: cheapest pre-book + the saving range (use existing engine helpers; no new data).
  Render via the existing `AnswerLead`/`MiniAnswerBar` components (or a small prose block) so it's
  extractable with JS off. Put the derivation in a tiny pure helper so it's unit-testable.
- **FAQ date/source:** in `apps/parkmath/lib/content.ts` `buildDropOffFaqs`, change the primary fee
  answer from bare `record.feeSummary` to `${record.feeSummary} (verified ${record.verifiedAt}, per the official ${airportName} page).`
- **Testing (unit):** the index-summary helper returns correct cheapest/dearest/free-count for a
  fixture; `buildDropOffFaqs` fee answer contains the verified date + "official".

### E. LCP font preload *(P1-1, quick win 4)*

- In `apps/parkmath/app/layout.tsx`, set `preload: false` on the **`IBM_Plex_Mono`** `next/font` call
  (mono renders numbers, not the LCP H1 — it can swap), keeping `IBM_Plex_Sans` preloaded so its
  600/700 weights enter the preload budget.
- **Verification (preview):** the document head preloads a Plex **Sans** 600/700 woff2 and no longer
  preloads the three mono weights. *If Sans 700 still isn't preloaded after this change*, add an
  explicit `<link rel="preload" as="font" type="font/woff2" crossorigin>` for the H1 weight as a
  fallback (decide at build time from the observed head).

### F. Accessibility: skip link + `/news` heading *(P1-6, P1-2, quick win 9)*

- **Skip link:** in `layout.tsx`, add `<main id="main">` and, as the first child of `<body>`,
  `<a href="#main" class="sr-only focus:not-sr-only focus:absolute focus:z-50 focus:m-2 focus:rounded focus:bg-brand-accent focus:px-3 focus:py-2 focus:text-white">Skip to content</a>`.
- **`/news` heading:** `packages/ui/src/news-card.tsx` — add an optional `headingLevel?: "h2" | "h3"`
  prop (default `"h3"`, preserving `LatestUpdates`' correct usage); the `/news` hub
  (`apps/parkmath/app/news/page.tsx`) passes `headingLevel="h2"` so the outline is H1→H2.
- **Testing (unit, `packages/ui`):** `NewsCard` renders an `<h2>` when `headingLevel="h2"`, `<h3>` by default.

### G. Dependency hygiene *(P1-10)*

- In `apps/parkmath/package.json` (and `apps/roammath/package.json` for parity), change
  `next`/`react`/`react-dom` from `"latest"` to caret ranges pinned to the **currently-installed**
  versions (read them from the lockfile at build time, e.g. `"next": "^16.x.y"`, `"react": "^19.x.y"`).
  No `pnpm install` re-resolution beyond what the lockfile already holds.
